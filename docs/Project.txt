# 1. SESSION-BASED BATCH DATA ENTRY

Excellent idea.

Instead of mixing uploads randomly, every upload belongs to a:

# Session ID

Example:

| Session ID  | Meaning              |
| ----------- | -------------------- |
| 2023_JS1_T1 | JS1 First Term 2023  |
| 2024_JS2_T2 | JS2 Second Term 2024 |

This prevents:

* class conflicts,
* duplicate imports,
* mixed term analysis.

---

# Recommended Upload Metadata

When uploading:

| Field      | Example |
| ---------- | ------- |
| Session    | 2024    |
| Class      | JS2     |
| Term       | 2nd     |
| Sheet Name | JS2_T2  |

Then internally generate:

```text id="9rpr0h"
2024_JS2_T2
```

That becomes your unique dataset identifier.

---

# Why This Is Smart

Because your system can now:

✅ isolate batches
✅ compare sessions
✅ rebuild student timelines
✅ avoid overwriting data
✅ filter by academic period

Very realistic architecture.
